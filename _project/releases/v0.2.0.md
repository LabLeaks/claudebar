# v0.2.0 — Router Support & Session Intelligence

## New Features

### Router support
Route Claude Code through non-Anthropic models (OpenRouter, DeepSeek, Qwen, etc.) via a built-in Go proxy. No external dependencies.

- Named router configs in `~/.config/claudebar/config.json` define provider, API key, and model slots
- Activate from menu (`⌥M → Router`) or CLI (`claudebar --router=<name>`)
- Config wizard TUI for creating new router configs
- Per-session token count and cost displayed in status bar
- Native proxy on port 3457 with Anthropic ↔ OpenRouter protocol translation
- Transformer pipeline: tool call ID fix, JSON repair, max token capping

### Session intelligence
Smart session resume that distinguishes claudebar sessions from raw Claude sessions.

- Won't re-adopt killed sessions — only live claudebar sessions auto-resume
- Raw Claude sessions can be adopted via startup picker
- Session ID captured at startup for reliable `--resume` across restarts
- Stale state files cleaned up by cross-referencing against live tmux sessions

### Deferred restarts
Feature and router toggles now queue changes instead of restarting immediately. A single restart applies all changes when the menu closes.

## Improvements

- Status bar shows router name + token/cost when routing (replaces peak hours display)
- Cleaner error messages for proxy failures (HTML responses, ECONNRESET)
- tmux and claude dependency checks at startup with clear error messages
- Help popup height increased (was getting clipped)
- Security: directory names in statusline settings are now JSON-escaped

## Bug Fixes

- **Session jumping** — no longer resumes wrong conversation when multiple sessions exist for the same project
- **Test isolation** — tests no longer migrate real user state files (HOME isolation)
- **Proxy ECONNRESET** — proxy no longer crashes when Claude Code closes connections unexpectedly

## Breaking Changes

- **Config location (macOS, automatic):** State files migrated from `~/Library/Application Support/claudebar/` to `~/.config/claudebar/`. Migration is automatic on first run — no user action required.

## Internal

- New `openrouter/` package: proxy server, protocol transformation, usage tracking
- Replaced external CCR (claude-code-router) dependency with native Go proxy
- Comprehensive test suite for router config validation, env var generation, session management
- Codified conventions in `_project/patterns/`: error handling, tmux conventions, session management, TUI style, file organization, API surface inventory
- MIT license added
